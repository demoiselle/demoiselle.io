Versão Final 1.2.0 Liberada!

Para projetos que estejam iniciando a utilização de assinaturas e utilizem Java 7 ou superior é recomendada a utilização do Demoiselle Signer que está em desenvolvimento e pode ser acompanhado no endereço https://github.com/demoiselle/signer/.

Novas Funcionalidades

Novo Mecanismo de Descoberta da Cadeia de Autoridades do Certificado

Em várias situações o componente precisa ter certeza que o certificado informado possui a cadeia de certificados das Autoridades Certificadoras (ICP Brasil) corretas, e para isso é necessário que o componente reconheça para efetivar a validação.

Antes da versão 1.2.0 somente os certificados gerados pelas Autoridades Certificadoras que estivessem dentro do componente (por isso ele precisava ser atualizado com maior frequência) eram reconhecidos, agora o componente também tenta encontrar a cadeia de Autoridades de outras maneiras, seguindo a seguinte ordem de descoberta:

  • Dentro do componente (como era feito nas versões anteriores)
  • Home do usuário
  • Busca em um endereço do Serpro
  • Busca em um endereço do ITI
Quando em nenhum destes locais possuir todos os certificados para montar a cadeia do certificado é que ele será considerado inválido. Outra situação que deve ser observada é que por mais que a cadeia tenha sido montada corretamente é necessário que o certficado esteja aderente a política escolhida (Ex: AD RB 2.2 - Asssinatura Digital com Referência Básica 2.2).

Atualização para Java 7

Agora a versão necessária para rodar o componente é a versão 7 do Java.

Utilização do Java 6

Devido a problemas em projetos em produção que utilizam a versão 6 do Java foi necessário fazer o downgrade da versão 7 e agora o componente é compatível com Java 6 ou superior.

Correção da validação da identificação da política (Hash e Links da Política)

Ao validar alguns tipos de assinatura com a versão anterior ocorria um erro.

Como Atualizar

Para testar a nova versão do componente certificate em seu projeto atualize suas dependencias do maven conforme as sugestões abaixo: 

<dependency>
	<groupId>br.gov.frameworkdemoiselle.component</groupId>
	<artifactId>demoiselle-certificate-core</artifactId>
	<version>1.2.0</version>
</dependency>
<dependency>
	<groupId>br.gov.frameworkdemoiselle.component</groupId>
	<artifactId>demoiselle-certificate-ca-icpbrasil</artifactId>
	<version>1.2.0</version>
</dependency>
<dependency>
	<groupId>br.gov.frameworkdemoiselle.component</groupId>
	<artifactId>demoiselle-certificate-signer</artifactId>
	<version>1.2.0</version>
</dependency>

Para outras partes do componente como Applet e Desktop também é possível atualizar para a versão 1.2.0.

No ambiente de desenvolvimento Eclipse após adicionar é importante atualizar as configurações do Maven clicando sobre o projeto com o botão direito do mouse, acessando as opções do Maven e por fim clicando em Update Project. Na tela que será aberta selecione o seu projeto e clique em OK.

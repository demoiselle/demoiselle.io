Mecanismo de Descoberta da Cadeia de Autoridades do Certificado

Em várias situações o componente precisa ter certeza que o certificado informado possui a cadeia de certificados das Autoridades Certificadoras (ICP Brasil) corretas, e para isso é necessário que o componente reconheça para efetivar a validação.

Nas versões antigas somente os certificados gerados pelas Autoridades Certificadoras que estivessem dentro do componente (por isso ele precisava ser atualizado com maior frequência) eram reconhecidos, agora o componente também tenta encontrar a cadeia de Autoridades de outras maneiras, seguindo a seguinte ordem de descoberta:

  • Dentro do componente (como era feito nas versões anteriores)
  • Home do usuário
  • Busca em um endereço do Serpro
  • Busca em um endereço do ITI
Quando em nenhum destes locais possuir todos os certificados para montar a cadeia do certificado é que ele será considerado inválido. Outra situação que deve ser observada é que por mais que a cadeia tenha sido montada corretamente é necessário que o certficado esteja aderente a política escolhida (Ex: AD RB 2.2 - Asssinatura Digital com Referência Básica 2.2).

Java 7

Para utilizar esta versão o Java 7 é obrigatório.

Como Utilizar

Para testar o componente Demoiselle Signer em seu projeto atualize suas dependencias do maven conforme as sugestões abaixo: 

<dependency>
	<groupId>org.demoiselle.signer</groupId>
	<artifactId>core</artifactId>
	<version>3.1.2</version>
</dependency>
<dependency>
	<groupId>org.demoiselle.signer</groupId>
	<artifactId>policy-impl-cades</artifactId>
	<version>3.1.2</version>
</dependency>
<dependency>
	<groupId>org.demoiselle.signer</groupId>
	<artifactId>chain-icp-brasil</artifactId>
	<version>3.1.2</version>
</dependency>
